# Informe colaborativo: revisión de código responsiva

**Recordatorio teórico** El diseño responsivo es la práctica de construir interfaces que se adaptan fluidamente a cualquier tamaño de pantalla, desde móviles hasta pantallas de escritorio. Los tres pilares fundamentales son: las media queries (puntos de ruptura que permiten aplicar estilos condicionales según el ancho del viewport), Flexbox (sistema unidimensional ideal para alinear y distribuir elementos en filas o columnas) y CSS Grid (sistema bidimensional que permite definir áreas y columnas/filas de forma explícita). La filosofía Mobile First implica escribir estilos base para pantallas pequeñas y sobrescribir progresivamente con media queries de tipo `min-width`. Errores frecuentes: usar `max-width` en todos los breakpoints (enfoque Desktop First sin ser consciente de ello), olvidar el meta viewport (``), mezclar unidades fijas (px) con unidades relativas (%, rem, vw) sin coherencia, y no probar en dispositivos reales o emulados. Un mini-ejemplo: si defines `display: grid; grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));` en un contenedor, obtienes un layout que se reorganiza solo sin una sola media query explícita. Comprender cuándo usar Grid, cuándo usar Flexbox y cuándo combinarlos es la clave para producir código responsivo limpio, mantenible y listo para producción. --- **Contexto del taller** Este taller cierra el capítulo de Diseño Responsivo con CSS3, Flexbox y Grid de una manera práctica y orientada al trabajo en equipo real: la revisión de código de otro desarrollador. En el mundo profesional, los code reviews son una práctica habitual en equipos ágiles. Saber leer código ajeno, identificar problemas de responsividad, proponer mejoras fundamentadas y redactar un informe técnico claro...

# Taller: hacer que una tabla sea accesible (alcance, resumen)

RECORDATORIO TEÓRICO Una tabla HTML es accesible cuando cualquier usuario, incluidos quienes utilizan lectores de pantalla, puede comprender la relación entre los datos de las celdas y sus encabezados. Para lograrlo, HTML ofrece atributos y elementos específicos que van más allá de la estructura básica vista anteriormente (table, thead, tbody, th, caption, scope). El atributo scope aplicado a los elementos indica si un encabezado corresponde a una columna (scope="col"), a una fila (scope="row"), a un grupo de columnas (scope="colgroup") o a un grupo de filas (scope="rowgroup"). Sin este atributo, un lector de pantalla no sabe a qué celda de datos pertenece el encabezado, lo que vuelve la tabla incomprensible para usuarios con discapacidad visual. El atributo summary (en HTML4/XHTML) y su equivalente moderno, el elemento acompañado de una descripción detallada o el atributo aria-describedby, proporcionan un resumen del propósito y la estructura de la tabla. En HTML5, summary está obsoleto; se recomienda usar para el título visible y aria-describedby apuntando a un párrafo descriptivo oculto o visible según el diseño. Errores frecuentes: usar en lugar de para encabezados, omitir scope en tablas complejas, no incluir , confundir el título de la tabla con su descripción funcional.

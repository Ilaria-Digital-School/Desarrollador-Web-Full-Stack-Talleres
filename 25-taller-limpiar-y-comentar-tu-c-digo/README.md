# Taller: limpiar y comentar tu código

**Recordatorio teórico** Un código HTML limpio y bien comentado no es un lujo: es una exigencia profesional. En un entorno de trabajo real, tu código será leído, mantenido y modificado por otros desarrolladores (o por ti mismo dentro de seis meses). La legibilidad es, por tanto, una competencia técnica en sí misma. La **sangría** (indentación) consiste en desplazar hacia la derecha, de forma coherente, el contenido anidado dentro de un elemento padre. El estándar más extendido es utilizar 2 o 4 espacios (jamás mezclar tabulaciones y espacios). Una sangría correcta permite visualizar de un vistazo la jerarquía del documento. Los **comentarios HTML** se escriben entre ``. Sirven para explicar la función de una sección, señalar una zona de trabajo en progreso o delimitar visualmente bloques importantes. Ojo: no se deben comentar cosas evidentes ni dejar comentarios obsoletos que ya no correspondan al código actual. Ese es uno de los errores más frecuentes de los principiantes. Un error habitual es también dejar **código muerto** (fragmentos comentados que nunca se eliminan), etiquetas sin cerrar o atributos duplicados. La denominación de archivos y la estructura de árbol, ya vistas anteriormente, forman parte de este mismo principio de orden y claridad. Modelo mental clave: imagina que tu código es un documento de trabajo que otra persona debe comprender en menos de dos minutos. Si no puede hacerlo, el código necesita ser limpiado. --- **Enunciado del taller** En este taller vas a trabajar sobre un archivo HTML intencionadamente desordenado.

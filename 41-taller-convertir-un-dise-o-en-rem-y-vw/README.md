# Taller: convertir un diseño en rem y vw

**Recordatorio teórico** Antes de comenzar el taller, repasa estos conceptos clave. Las unidades relativas en CSS permiten crear diseños que se adaptan a diferentes tamaños de pantalla sin depender de valores fijos en píxeles. El `rem` (root em) es relativo al tamaño de fuente del elemento raíz (``); por defecto, `1rem = 16px`. Esto lo hace ideal para tipografía, espaciados y tamaños de componentes, ya que el usuario puede ajustar la fuente base del navegador y todo escala proporcionalmente. El `vw` (viewport width) equivale al 1% del ancho de la ventana gráfica; por tanto, `100vw` ocupa el ancho completo de la pantalla. Es ideal para anchos de contenedores, tamaños de fuente fluidos y elementos que deben escalar con la pantalla. El error más frecuente entre principiantes es mezclar `px` y `rem` sin criterio, o aplicar `vw` a propiedades donde no aporta valor real (como el `padding` vertical). También es habitual olvidar definir el tamaño base en el selector `html`, lo que genera comportamientos inesperados. Otro error clásico: usar `vw` para fuentes sin límite mínimo, lo que produce textos ilegibles en pantallas pequeñas. Mini-ejemplo: si tienes `font-size: 24px` en un título, la conversión a `rem` es `24 / 16 = 1.5rem`. Para un contenedor de `960px` de ancho en una pantalla de `1200px`, la conversión a `vw` sería `(960 / 1200) * 100 = 80vw`.

# Miniproyecto final: portafolio de varias páginas completamente semántico

## Descripción del proyecto Este miniproyecto de cierre de capítulo te invita a construir un **portafolio personal de varias páginas** aplicando todas las competencias de HTML semántico adquiridas a lo largo del capítulo. El sitio deberá estar estructurado de forma rigurosa, accesible y semánticamente correcta, sin necesidad de un framework externo: solo HTML y CSS. El portafolio representará tu identidad como desarrollador web en formación y servirá como base real que podrás enriquecer a lo largo del curso. ## Competencias que se validan - Uso correcto de las etiquetas estructurales (`header`, `nav`, `main`, `section`, `article`, `aside`, `footer`). - Uso de etiquetas de texto en línea semánticas (`time`, `abbr`, `code`, `strong`, `em`, `mark`, `cite`, etc.). - Creación de tablas semánticas accesibles (`table`, `thead`, `tbody`, `tfoot`, `th`, `caption`, `scope`). - Estructuración semántica de extremo a extremo coherente entre páginas. - Aplicación de buenas prácticas de accesibilidad (atributos `alt`, `aria-label`, `scope`, `summary`, navegación por teclado). - Uso de metadatos HTML (`meta`, `title`, `lang`, `charset`) y estructura de documento completa. - Jerarquía correcta de encabezados (`h1` a `h6`). - Organización de archivos HTML y CSS en un proyecto multipágina. ## Contexto profesional Como desarrollador web Full Stack, serás responsable de estructurar interfaces web que sean comprensibles tanto para los navegadores como para los motores de búsqueda y las tecnologías de asistencia. Este portafolio es el primer entregable profesional real de tu formación.

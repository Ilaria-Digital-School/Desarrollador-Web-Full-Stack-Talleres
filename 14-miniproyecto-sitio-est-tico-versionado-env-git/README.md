# Miniproyecto: sitio estático versionado (env + Git)

## Descripción del proyecto Este mini-proyecto tiene como objetivo consolidar todos los conocimientos adquiridos en el capítulo introductorio sobre el entorno de desarrollo y Git. El estudiante deberá crear un sitio web estático completo (HTML/CSS) y gestionarlo con Git desde el inicio, aplicando las buenas prácticas de versionado aprendidas durante el curso. El proyecto consiste en construir un sitio web personal de presentación (tipo portfolio o página de presentación profesional) compuesto por al menos 3 páginas HTML, con estilos CSS organizados, versionado con Git desde el primer commit, y correctamente configurado con un archivo .gitignore adaptado. ## Competencias validadas - Configurar y utilizar un entorno de desarrollo local profesional - Inicializar y gestionar un repositorio Git local - Redactar mensajes de commit claros, descriptivos y siguiendo las buenas prácticas - Crear y gestionar ramas (branches) en Git - Utilizar un archivo .gitignore adaptado al proyecto - Consultar e interpretar el historial de commits con git log - Organizar el trabajo en commits coherentes y progresivos ## Contexto profesional Como desarrollador web, cada proyecto que realices deberá estar versionado desde el primer día. Este mini-proyecto simula las condiciones reales de trabajo: arrancar un proyecto desde cero, estructurarlo correctamente y mantener un historial de versiones limpio y profesional, habilidades fundamentales antes de abordar JavaScript, Node.js y el trabajo en equipo.

# Taller: implementación de un potente conjunto de imágenes adaptables

RECORDATORIO TEÓRICO Las imágenes adaptables (responsive images) son uno de los pilares del rendimiento web moderno. Su objetivo es servir la imagen más adecuada según el dispositivo del usuario, evitando descargar archivos innecesariamente pesados en pantallas pequeñas o de baja resolución. Dos atributos HTML son fundamentales: srcset y sizes. El atributo srcset permite declarar múltiples versiones de una misma imagen con sus respectivos anchos (descriptores w) o densidades de píxeles (descriptores x). El atributo sizes describe al navegador qué ancho ocupará la imagen en pantalla según condiciones de media query, para que pueda elegir la fuente óptima antes de renderizar la página. El elemento va más lejos: permite servir formatos distintos (WebP, AVIF) o composiciones visuales diferentes (art direction) según el contexto. La carga diferida (lazy loading) con el atributo loading='lazy' complementa este sistema posponiendo la descarga de imágenes fuera del viewport. Errores frecuentes: confundir srcset con sizes (el primero declara las opciones disponibles, el segundo describe el uso real en pantalla); omitir el atributo src de respaldo para navegadores antiguos; usar solo densidades de píxeles (1x, 2x) cuando los descriptores de anchura (w) son más eficientes en la mayoría de casos.

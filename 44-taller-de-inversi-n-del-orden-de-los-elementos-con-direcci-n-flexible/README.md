# Taller de : inversión del orden de los elementos con dirección flexible

**Recordatorio teórico** Flexbox ofrece una propiedad esencial para controlar la dirección del flujo de los elementos dentro de un contenedor flexible: `flex-direction`. Por defecto, su valor es `row`, lo que dispone los elementos en fila de izquierda a derecha. Sin embargo, existen cuatro valores posibles: `row` (izquierda a derecha), `row-reverse` (derecha a izquierda), `column` (de arriba a abajo) y `column-reverse` (de abajo a arriba). Los valores con el sufijo `-reverse` son especialmente útiles cuando se desea invertir visualmente el orden de los elementos sin modificar el orden del HTML, lo cual es una buena práctica para mantener la accesibilidad y la semántica del DOM intactas. El modelo mental clave es el siguiente: `flex-direction` define el **eje principal** del contenedor. Cuando se invierte la dirección, el eje principal también se invierte, lo que impacta directamente en el comportamiento de propiedades como `justify-content` y `align-items`. Error frecuente: confundir el orden visual con el orden real del DOM. Recuerda que `row-reverse` no modifica el HTML, solo el renderizado visual. Mini-ejemplo: si tienes tres cajas con las clases `.caja-1`, `.caja-2`, `.caja-3` dentro de un contenedor `.flex-container { display: flex; flex-direction: row-reverse; }`, visualmente aparecerán en orden 3, 2, 1, pero en el DOM siguen siendo 1, 2, 3. **Enunciado del taller** En este taller vas a construir una pequeña interfaz responsiva que demuestre, de forma práctica y visible, el comportamiento de la propiedad `flex-direction` con sus cuatro valores, haciendo especial énfasis en los valores de inversión (`row-reverse` y `column-reverse`).

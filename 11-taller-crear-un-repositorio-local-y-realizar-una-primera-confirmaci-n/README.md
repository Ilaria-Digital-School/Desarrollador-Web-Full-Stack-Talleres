# Taller: crear un repositorio local y realizar una primera confirmación

**Recordatorio teórico** Git es un sistema de control de versiones distribuido que permite registrar el historial de cambios de un proyecto a lo largo del tiempo. La unidad fundamental de trabajo en Git es el repositorio (repo): una carpeta que contiene tu código y una carpeta oculta .git donde Git almacena todo el historial. El flujo básico de Git se articula en tres zonas: el directorio de trabajo (donde editas tus archivos), el área de preparación o staging area (donde seleccionas qué cambios incluirás en la próxima confirmación) y el repositorio local (donde se guardan definitivamente las confirmaciones). Una confirmación o commit es una instantánea del estado de tu proyecto en un momento dado: tiene un identificador único (hash), un autor, una fecha y un mensaje descriptivo. El mensaje de commit es crucial: debe ser claro y conciso, describiendo QUÉ cambia y POR QUÉ. Un error frecuente de los principiantes es hacer commits con mensajes vacíos o demasiado vagos como «cambios» o «arreglos». Otro error habitual es olvidar pasar los archivos por el staging area antes de confirmar. Mini-ejemplo mental: imagina que Git es una cámara fotográfica. git add selecciona lo que entra en el encuadre; git commit dispara la foto y la guarda para siempre en el álbum. **Objetivos del taller** Este taller tiene como objetivo consolidar los conocimientos introductorios sobre Git aprendidos en el capítulo anterior.

# Taller: inspección de una página HTML con DevTools

RECORDATORIO TEÓRICO Las DevTools (herramientas de desarrollo del navegador) son una suite integrada en todos los navegadores modernos (Chrome, Firefox, Edge) que permite inspeccionar, depurar y analizar cualquier página web en tiempo real. Son la navaja suiza del desarrollador web: sin ellas, trabajar en front-end equivaldría a construir una casa con los ojos cerrados. Conceptos clave que debes tener claros antes de empezar: - El panel «Elements» (o «Inspector» en Firefox) muestra el árbol DOM de la página, es decir, la estructura HTML tal como el navegador la interpreta en ese momento, que puede diferir del archivo fuente original si JavaScript ha modificado el contenido dinámicamente. - El panel «Console» permite leer mensajes de error, advertencias y ejecutar código JavaScript directamente en el contexto de la página. - El panel «Network» registra todas las solicitudes HTTP realizadas por la página (imágenes, scripts, hojas de estilo, llamadas API). Error frecuente: confundir el código fuente original (Ctrl+U) con el DOM inspeccionado en DevTools. El DOM es una representación viva y modificable; el código fuente es el archivo tal como llegó del servidor. Mini-ejemplo mental: imagina una página con un botón que cambia de color al hacer clic. En el código fuente, ese botón siempre será rojo. En DevTools, tras el clic, verás que su atributo class o style ha cambiado en tiempo real.

# Taller: inspección de una página HTML con DevTools

Recordatorio teórico (≈190 palabras) Inspectores de navegador —en particular DevTools de Chrome o Firefox— son paneles integrados que permiten observar y modificar temporalmente la estructura DOM, los estilos CSS, la red y la consola JavaScript de cualquier página web. Piensa en DevTools como un microscopio de capas: la pestaña Elements revela el árbol vivo del DOM; Styles muestra las reglas aplicadas y su cascada; Network fotografía cada petición; Console ejecuta y depura código. Error frecuente: confundir el HTML original descargado con el DOM actualizado: cuando JavaScript inyecta nodos, el código mostrado en Elements ya no coincide con el archivo .html servido. Otro tropiezo habitual es editar un nodo sin recordar que el cambio sólo vive mientras la página no se recargue. Mini-ejemplo: abre devtools (F12), selecciona el logo de Google con el cursor inspector, cambia su atributo src en Elements y observa la sustitución instantánea: compruebas así el ciclo inspeccionar-modificar-ver resultado. Enunciado general En este taller analizarás una página pública real y redactarás un informe técnico que describa su estructura semántica, jerarquía de estilos y comportamiento de carga. Trabajarás en solitario o en pareja, dispondrás de 3 horas y deberás entregar un documento (Markdown o PDF) y un archivo .har antes de que el tiempo expire. Se evaluarán tu método de investigación, la claridad de tus observaciones y tu capacidad para relacionar hallazgos con buenas prácticas vistas en clase.

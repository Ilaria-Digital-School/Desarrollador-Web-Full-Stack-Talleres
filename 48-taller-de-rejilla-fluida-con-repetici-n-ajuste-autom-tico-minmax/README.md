# Taller de : rejilla fluida con repetición (ajuste automático, minmax)

**Recordatorio teórico** CSS Grid ofrece una función muy poderosa para crear rejillas fluidas sin necesidad de media queries explícitas: la combinación de `repeat()`, `auto-fill` o `auto-fit`, y `minmax()`. Comprender cada pieza es fundamental antes de escribir una sola línea de código. `repeat(n, tamaño)` permite definir columnas repetidas sin escribirlas una a una. Cuando se sustituye `n` por `auto-fill`, el navegador crea tantas columnas como quepan en el contenedor según el tamaño mínimo indicado, aunque queden columnas vacías. Con `auto-fit`, las columnas vacías se colapsan y el espacio sobrante se distribuye entre los ítems existentes: esto es lo que generalmente se prefiere para galerías o tarjetas de contenido. `minmax(min, max)` define un rango de tamaño para cada columna: nunca será más pequeña que `min` ni más grande que `max`. Usar `1fr` como valor máximo garantiza que las columnas se expandan para llenar el espacio disponible. El patrón clásico es: `grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));`. Este fragmento crea una rejilla adaptativa donde cada columna tiene al menos 250 px y crece fluidamente. **Error frecuente**: confundir `auto-fill` con `auto-fit`. Recuerda: `auto-fill` mantiene las columnas vacías (útil cuando el contenedor necesita una anchura fija preservada), mientras que `auto-fit` las elimina visualmente. Mini-ejemplo: si tienes 3 ítems en un contenedor de 900 px y cada columna tiene un mínimo de 200 px, con `auto-fit` los 3 ítems se estiran para cubrir todo el ancho; con `auto-fill` se crean columnas fantasma y los ítems no se expanden más allá de su tamaño calculado. --- **Enunciado del taller**...

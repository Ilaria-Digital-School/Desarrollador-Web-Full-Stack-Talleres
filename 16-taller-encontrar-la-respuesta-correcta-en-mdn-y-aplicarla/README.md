# Taller: encontrar la respuesta correcta en MDN y aplicarla

**Recordatorio teórico** MDN Web Docs (Mozilla Developer Network) es la referencia técnica oficial para los estándares web: HTML, CSS y JavaScript. A diferencia de un tutorial de blog o un hilo de StackOverflow, MDN describe el comportamiento esperado de cada elemento, propiedad o función según los estándares del W3C y WHATWG. La estructura típica de una página MDN incluye: una descripción conceptual, la sintaxis formal, los parámetros o atributos aceptados, ejemplos de código interactivos, notas de compatibilidad entre navegadores y una sección 'Véase también' con referencias cruzadas. Un error frecuente entre los principiantes es buscar directamente en Google frases como '¿cómo centrar un div?' y aplicar el primer fragmento de código que aparece sin entender por qué funciona. Esto genera dependencia de soluciones copiadas y dificulta la depuración posterior. El modelo mental correcto es diferente: identificar el concepto técnico implicado (por ejemplo, 'flexbox', 'position', 'display'), buscar ese concepto en MDN, leer la descripción y la sintaxis, y solo entonces aplicar el ejemplo al propio código. Mini-ejemplo: si un elemento HTML no ocupa el ancho esperado, el reflejo correcto no es buscar 'cómo hacer que un div sea más ancho', sino ir a MDN, buscar la propiedad `width` o `box-sizing`, leer cómo interactúan ambas con el modelo de caja, y aplicar la solución con comprensión.

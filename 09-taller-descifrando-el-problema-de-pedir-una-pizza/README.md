# Taller: descifrando el problema de «pedir una pizza»

**Recordatorio teórico** La mentalidad del desarrollador no consiste únicamente en escribir código: consiste ante todo en descomponer un problema complejo en pasos pequeños, ordenados y ejecutables. Este proceso se llama análisis de problemas o decomposición algorítmica. Cuando un programa falla, cuando una funcionalidad no funciona o cuando hay que construir algo desde cero, el primer reflejo del desarrollador profesional es preguntarse: ¿cuál es la entrada? ¿Cuál es la salida esperada? ¿Qué pasos intermedios son necesarios? Un error frecuente en principiantes es querer resolver todo de golpe, sin estructurar el razonamiento. Esto genera bloqueos, código desordenado y dificultad para depurar. La solución es sistemática: escribir los pasos en lenguaje natural antes de escribir cualquier línea de código. A este enfoque se le llama pseudocódigo o diagrama de flujo mental. Mini-ejemplo: para «hacer café», no escribimos directamente código. Primero listamos: (1) verificar que hay café, (2) poner agua en la cafetera, (3) encender la cafetera, (4) esperar, (5) servir. Cada paso es concreto, ordenado y verificable. Esta misma lógica se aplica a cualquier algoritmo o funcionalidad web. La acción de «pedir una pizza» en una aplicación real implica múltiples actores, decisiones y validaciones que, si no se descomponen correctamente, generan errores difíciles de detectar. --- **Enunciado del taller** En los talleres y cuestionarios anteriores has aprendido a usar DevTools, a observar errores en la consola y a desarrollar una mentalidad analítica frente a los problemas. Este taller te propone dar un paso más: aplicar esa mentalidad antes de escribir cualquier código, trabajando...

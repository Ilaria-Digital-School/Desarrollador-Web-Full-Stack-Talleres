# Taller: construcción del modelo de un blog de una columna

**Recordatorio teórico** HTML5 introdujo un conjunto de etiquetas semánticas cuyo propósito es describir el significado del contenido que contienen, no simplemente su apariencia visual. Esto permite que los navegadores, motores de búsqueda y tecnologías de asistencia (como los lectores de pantalla) interpreten correctamente la estructura de una página web. Las etiquetas fundamentales para estructurar una página de tipo blog son: (encabezado del sitio, generalmente contiene el logo y la navegación principal), (menú de navegación), (contenido principal y único de la página), (agrupación temática de contenido), (bloque de contenido independiente y reutilizable, ideal para entradas de blog), (contenido complementario o lateral, aunque en un blog de una columna puede estar integrado dentro del flujo principal), y (pie de página con información de contacto, derechos de autor, etc.). Un error frecuente es seguir utilizando genéricos cuando existe una etiqueta semántica apropiada, o anidar etiquetas de forma ilógica (por ejemplo, colocar un dentro de un sin intención semántica clara). Otro error común es confundir y : usa cuando el bloque tiene sentido por sí solo fuera de contexto, y para agrupar contenido relacionado dentro de una misma temática. Mini-ejemplo: una entrada de blog es un que contiene un con el título y la fecha, luego los párrafos del cuerpo, y finalmente un con el autor.

# Taller guiado: identificación de componentes (editor, terminal, navegador)

Recordatorio teórico (≈190 palabras). Un entorno de desarrollo local básico se apoya siempre en tres piezas: 1) un editor de código, donde escribes y organizas los archivos fuente; 2) una terminal, que permite lanzar comandos del sistema y de tus herramientas; y 3) un navegador, que renderiza la interfaz y expone la consola del desarrollador. El modelo mental que debes adoptar es el de un circuito cerrado: escribes en el editor, ejecutas-compilas-sirves desde la terminal y observas el resultado en el navegador, volviendo al editor para iterar. Tres errores frecuentes en principiantes son: a) confundir el editor con el IDE completo y pretender que haga todo por sí solo; b) temer a la terminal y caer en la dependencia exclusiva de interfaces gráficas; c) probar el código únicamente con doble clic sobre un archivo HTML, sin pasar por un servidor local. Mini-ejemplo: abres Visual Studio Code (editor), lanzas `npx http-server` en PowerShell (terminal) dentro de la carpeta del proyecto y, finalmente, visitas `http://localhost:8080` en Chrome (navegador) mientras inspeccionas la consola. Con estas bases claras, el taller pondrá a prueba tu capacidad para identificar y configurar estos tres componentes desde cero. Enunciado general. Vas a instalar o verificar la presencia de un editor moderno, una terminal funcional y un navegador actualizado, documentar cada paso y justificar tus elecciones. El resultado final será un pequeño informe en formato Markdown acompañado de capturas de pantalla y un proyecto de demostración mínimamente interactivo.

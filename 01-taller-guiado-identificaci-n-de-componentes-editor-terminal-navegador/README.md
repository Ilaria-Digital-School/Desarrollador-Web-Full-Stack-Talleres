# Taller guiado: identificación de componentes (editor, terminal, navegador)

**Recordatorio teórico** Un entorno de desarrollo es el conjunto de herramientas que un desarrollador utiliza para escribir, ejecutar, probar y depurar código. Está compuesto por tres pilares fundamentales que conviene distinguir con claridad: - **El editor de código** (por ejemplo, Visual Studio Code) es el espacio donde se escribe y organiza el código fuente. Ofrece funciones como resaltado de sintaxis, autocompletado y detección de errores en tiempo real. No es un procesador de texto: está diseñado exclusivamente para trabajar con código. - **La terminal** (también llamada consola o línea de comandos) es la interfaz de texto que permite comunicarse directamente con el sistema operativo: navegar por carpetas, ejecutar scripts, instalar dependencias o lanzar un servidor. Dominarla es una habilidad fundamental para cualquier desarrollador. - **El navegador web** (por ejemplo, Chrome o Firefox) es el entorno donde se ejecuta y se visualiza el código front-end. Sus herramientas de desarrollador integradas (DevTools) permiten inspeccionar el DOM, depurar JavaScript y analizar el rendimiento. Error frecuente: confundir el editor con el navegador, o creer que la terminal es opcional. En la práctica profesional, estos tres componentes se utilizan de forma conjunta y complementaria en cada sesión de trabajo. Mini-ejemplo: cuando un desarrollador crea un archivo `index.html` en VS Code (editor), lo abre en Chrome (navegador) para ver el resultado visual, y usa la terminal para ejecutar `git commit` y guardar su progreso. --- **Enunciado del taller** Este taller tiene como objetivo consolidar tu comprensión práctica de los tres componentes esenciales del entorno de desarrollo:...

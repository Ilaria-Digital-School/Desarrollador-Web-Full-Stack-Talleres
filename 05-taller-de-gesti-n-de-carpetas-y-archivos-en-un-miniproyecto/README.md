# Taller de : gestión de carpetas y archivos en un miniproyecto

**Recordatorio teórico** Antes de comenzar el taller, repasa estos conceptos esenciales. La terminal es una interfaz de texto que permite interactuar directamente con el sistema operativo mediante comandos. A diferencia del explorador de archivos gráfico, la terminal te da velocidad, precisión y control total sobre tu entorno de trabajo, algo imprescindible para cualquier desarrollador. Un sistema de archivos se organiza de forma jerárquica: existe un directorio raíz desde el cual se ramifican carpetas (directorios) y archivos. Cada archivo o carpeta tiene una ruta absoluta (desde la raíz) y una ruta relativa (desde tu posición actual). Saber navegar y estructurar esta jerarquía es la base de todo proyecto de software. Los comandos clave que ya conoces son: `cd` (cambiar de directorio), `ls` (listar contenido), `mkdir` (crear carpeta), `touch` (crear archivo vacío). Un error frecuente de los principiantes es crear archivos y carpetas sin planificar la estructura, lo que genera proyectos desordenados difíciles de mantener. Mini-ejemplo mental: imagina que tu proyecto es una casa; cada carpeta es una habitación con un propósito claro. No pondrías la ropa en la cocina. Este taller te propone poner en práctica todo lo aprendido creando desde cero la estructura de archivos de un miniproyecto web real, únicamente usando la terminal. --- **Enunciado del taller** Tu objetivo en este taller es construir, exclusivamente desde la terminal, la estructura completa de carpetas y archivos de un miniproyecto web llamado `mi-portfolio`. Este proyecto simulará la arquitectura inicial de un sitio web personal básico, tal como lo haría un desarrollador...

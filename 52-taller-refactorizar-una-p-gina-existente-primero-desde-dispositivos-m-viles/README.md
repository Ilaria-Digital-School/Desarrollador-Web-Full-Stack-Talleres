# Taller: refactorizar una página existente primero desde dispositivos móviles

**Recordatorio teórico** El enfoque «mobile-first» consiste en escribir primero los estilos CSS para pantallas pequeñas (móviles) y luego ampliarlos progresivamente mediante media queries con `min-width` para pantallas más grandes (tabletas, escritorios). Es la inversión del enfoque clásico «desktop-first», que usaba `max-width` para reducir el diseño. El modelo mental clave es el siguiente: el CSS base representa el estado mínimo y más simple del diseño; cada media query añade complejidad, no la elimina. Esto produce código más limpio, más ligero y más fácil de mantener. Definiciones esenciales: una media query con `min-width: 768px` se activa cuando la pantalla tiene al menos 768 px de ancho; los breakpoints típicos son 480 px (móvil grande), 768 px (tableta) y 1024 px (escritorio). Flexbox gestiona la distribución en un eje (fila o columna), mientras que Grid gestiona dos ejes simultáneamente; ambos se complementan perfectamente en un diseño responsivo. Errores frecuentes al refactorizar: olvidar eliminar las reglas `max-width` del código original antes de añadir las de `min-width`, lo que provoca conflictos; no restablecer propiedades heredadas del diseño desktop que rompen el layout móvil; y aplicar `display: grid` o `display: flex` sin adaptar la dirección y el número de columnas por breakpoint. Mini-ejemplo: en mobile-first, el contenedor principal parte con `display: flex; flex-direction: column;` y, a partir de 768 px, se convierte en `flex-direction: row;`.

# Taller: agregar un archivo.gitignore adaptado a un proyecto HTML/CSS

**Recordatorio teórico** El archivo .gitignore es un archivo de texto plano ubicado en la raíz de tu repositorio Git que le indica a Git qué archivos o carpetas debe ignorar y, por tanto, no incluir en el seguimiento de versiones. Su utilidad es fundamental: evita que archivos innecesarios, sensibles o generados automáticamente contaminen el historial de confirmaciones del proyecto. Un error frecuente de los principiantes es confirmar (commit) archivos como configuraciones locales del editor, carpetas de dependencias, archivos del sistema operativo (.DS_Store en macOS, Thumbs.db en Windows) o archivos temporales que no aportan ningún valor al proyecto compartido. Cada línea del .gitignore representa un patrón: puedes ignorar un archivo concreto (mi-archivo.txt), una carpeta entera (node_modules/), o todos los archivos con una extensión determinada (*.log). Un asterisco (*) actúa como comodín. Es importante saber que el .gitignore solo funciona sobre archivos que aún no han sido rastreados por Git; si ya confirmaste un archivo por error, deberás eliminarlo del índice con git rm --cached antes de que el .gitignore surta efecto. Por ejemplo, para un proyecto HTML/CSS básico, querrías ignorar la carpeta .vscode/ (configuración local del editor), archivos .DS_Store y cualquier archivo temporal generado por herramientas de tu entorno local. --- **Contexto del taller** Acabas de aprender qué es un archivo .gitignore y por qué es una práctica esencial en todo proyecto gestionado con Git.

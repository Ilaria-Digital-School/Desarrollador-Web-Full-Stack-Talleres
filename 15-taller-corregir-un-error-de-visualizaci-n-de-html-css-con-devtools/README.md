# Taller: corregir un error de visualización de HTML/CSS con DevTools

**Recordatorio teórico** Las DevTools (herramientas de desarrollador) son un conjunto de utilidades integradas en los navegadores modernos (Chrome, Firefox, Edge) que permiten inspeccionar, depurar y modificar en tiempo real el HTML y el CSS de cualquier página web. Se accede a ellas con F12 o clic derecho → 'Inspeccionar'. El panel más relevante para el trabajo de visualización es el Inspector de elementos: a la izquierda muestra el árbol DOM (la estructura HTML renderizada), y a la derecha el panel de estilos CSS aplicados a cada elemento. Un modelo mental clave: el navegador aplica los estilos en cascada. Esto significa que una regla puede ser sobrescrita por otra más específica o más tardía. En DevTools, las reglas tachadas indican propiedades anuladas. Los errores de visualización más frecuentes son: un selector CSS mal escrito o demasiado poco específico, una propiedad heredada no deseada, un margen o padding inesperado, un elemento en display incorrecto (por ejemplo, inline en lugar de block), o una clase HTML que no coincide exactamente con la definida en el CSS. Mini-ejemplo: si un botón no aparece centrado, basta con seleccionarlo en el Inspector, verificar qué regla de margin o text-align está activa, si alguna está tachada, y modificarla directamente en el panel de estilos para confirmar la corrección antes de editarla en el código fuente. --- **Enunciado del taller** En este taller pondrás en práctica el método de depuración paso a paso aplicado específicamente a errores de visualización HTML/CSS.

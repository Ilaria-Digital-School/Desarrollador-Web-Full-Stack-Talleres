# Laboratorio: instalación de Git y configuración de user.name/user.email

**Recordatorio teórico** Git es un sistema de control de versiones distribuido creado por Linus Torvalds en 2005. Su función principal es registrar el historial de cambios de un proyecto, permitiendo volver a versiones anteriores, colaborar con otros desarrolladores y mantener un seguimiento preciso de quién modificó qué y cuándo. A diferencia de guardar copias manuales de archivos, Git gestiona instantáneas del proyecto completo de forma eficiente. Dos conceptos clave para este taller: el **repositorio** (la carpeta donde Git registra el historial) y la **configuración global** (ajustes que aplican a todos los proyectos de tu máquina). La configuración de `user.name` y `user.email` es obligatoria: Git la usa para firmar cada commit con tu identidad. Sin esta configuración, Git puede rechazar tus commits o asignarles una identidad incorrecta, lo que genera problemas graves en proyectos colaborativos. Error frecuente: instalar Git y empezar a trabajar sin configurar `user.name` ni `user.email`. Esto provoca commits anónimos o con datos erróneos que son difíciles de corregir después en un historial compartido. Mini-ejemplo mental: imagina Git como un notario que certifica cada cambio que haces en tu código. Para que el acta sea válida, el notario necesita saber tu nombre y tu correo. Sin esa información, el acta queda inválida o incompleta. --- **Enunciado del taller** En este taller pondrás en práctica los conceptos introducidos en el capítulo «Introducción a Git y al control de versiones».

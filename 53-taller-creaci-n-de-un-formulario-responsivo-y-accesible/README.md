# Taller: creación de un formulario responsivo y accesible

**Recordatorio teórico** Un formulario HTML accesible y responsivo combina tres disciplinas que has trabajado hasta ahora: diseño responsivo con CSS Grid/Flexbox, media queries con enfoque mobile-first y principios de accesibilidad web. Un formulario responsivo adapta su disposición según el tamaño de pantalla: en móvil, los campos suelen apilarse en una sola columna; en escritorio, pueden distribuirse en varias columnas usando Grid o Flexbox. La accesibilidad en formularios implica asociar correctamente cada `` a su `` mediante el atributo `for`/`id`, proporcionar textos de ayuda con `aria-describedby`, indicar los campos obligatorios con `required` y `aria-required`, y garantizar un contraste de color suficiente (mínimo 4.5:1 para texto normal). El error más frecuente es omitir los `` visibles y confiar únicamente en el atributo `placeholder`, que desaparece al escribir y no es suficiente para lectores de pantalla. Otro error habitual es no gestionar el estado de foco visible (`:focus-visible`), lo que penaliza la navegación por teclado. Mini-ejemplo: `Correo electrónicoFormato: nombre@dominio.com`. Recuerda aplicar siempre el enfoque mobile-first: define primero los estilos para pantallas pequeñas y amplía con media queries. --- **Enunciado del taller** En este taller pondrás en práctica de forma integrada todos los conceptos trabajados en la sección de diseño responsivo: Grid, Flexbox, media queries mobile-first y principios de accesibilidad. El objetivo concreto es construir desde cero un formulario de contacto responsivo y accesible que funcione correctamente en dispositivos móviles, tabletas y escritorio, y que respete los estándares básicos de accesibilidad web (WCAG 2.1 nivel AA).

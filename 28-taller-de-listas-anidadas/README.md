# Taller de : listas anidadas

**Recordatorio teórico: Listas anidadas en HTML** Una lista anidada es simplemente una lista que contiene otra lista en su interior. En HTML, esto se consigue colocando un elemento `` o `` directamente dentro de un `` de la lista padre, antes del cierre de esa etiqueta ``. Este mecanismo permite representar jerarquías de información: menús de navegación, estructuras organizacionales, tablas de contenido o, en el contexto de un currículum, secciones con subapartados detallados. Modelo mental clave: piensa en una lista anidada como un árbol. El tronco es la lista principal; cada rama es un ``; y de algunas ramas nacen sub-ramas, que son las listas hijas. Cada nivel de anidación debe reflejar una relación semántica real (padre → hijo), no simplemente un estilo visual. Errores frecuentes a evitar: abrir una `` hija directamente dentro de `` sin pasar por un `` — eso es HTML inválido; olvidar cerrar correctamente cada `` y `` en el orden inverso al que fueron abiertos; mezclar `` y `` sin justificación semántica (usa `` solo cuando el orden importa); y descuidar la indentación, que hace el código ilegible y dificulta la depuración.

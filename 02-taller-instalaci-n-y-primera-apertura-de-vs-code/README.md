# Taller: instalación y primera apertura de VS Code

Recordatorio teórico (≈180 palabras) El entorno de desarrollo integrado (IDE) es el punto de encuentro entre tu código y la máquina. VS Code pertenece a la familia de editores extensibles: ligero por defecto, potente gracias a la instalación de extensiones. Su modelo mental se basa en "abrir carpetas" en lugar de "abrir archivos sueltos"; esto favorece la visión de proyecto y la gestión de dependencias. Tres zonas concentran el 90 % de la actividad diaria: el editor, la barra lateral (explorador, control de código fuente, extensiones) y el terminal integrado. Error frecuente nº 1: instalar VS Code sin verificar la variable PATH, lo que impide lanzar el comando code desde la terminal. Error frecuente nº 2: abrir un subdirectorio "src" y luego preguntarse por qué Git o un linter no detectan el proyecto completo. Mini-ejemplo: al abrir la carpeta "mi-primera-web" en VS Code, el editor reconoce automáticamente el archivo package.json y propone instalar dependencias NPM; si solo abres index.html, esta detección se pierde. Mantén siempre la jerarquía de carpetas en mente. Enunciado del workshop Tu misión es instalar Visual Studio Code en tu sistema operativo principal (Windows, macOS o Linux) y realizar la primera apertura de un proyecto vacío siguiendo las buenas prácticas vistas en el capítulo anterior. El objetivo es que verifiques la correcta integración del editor con el sistema de archivos y la terminal, y que dejes tu entorno listo para los talleres posteriores.

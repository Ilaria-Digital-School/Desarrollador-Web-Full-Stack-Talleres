# Taller: provocar y leer un error de HTML/CSS

**Recordatorio teórico** En el desarrollo web, un error no es un fracaso: es información. HTML y CSS son lenguajes interpretados por el navegador, y cuando algo no funciona como se espera, el navegador intenta igualmente renderizar la página, a veces silenciando el problema. Por eso, aprender a provocar errores de forma controlada es una habilidad fundamental: te permite entender qué ocurre bajo el capó. Un error de HTML puede ser estructural (etiqueta no cerrada, atributo mal escrito, jerarquía rota) o semántico (usar un elemento donde no corresponde). Un error de CSS puede ser una propiedad inexistente, un valor incorrecto, un selector que no apunta a ningún elemento, o una regla que es anulada por otra de mayor especificidad. El navegador no siempre lanza un mensaje de error visible en pantalla: para verlos, debes abrir la consola y la pestaña Elements de las DevTools. Modelo mental clave: cuando algo no se muestra o no se aplica, pregúntate siempre «¿el navegador recibió el código? ¿Lo interpretó correctamente? ¿Hay algo que lo sobreescriba?». Errores frecuentes de principiantes: olvidar cerrar una etiqueta ``, escribir `colour` en lugar de `color`, referenciar un archivo CSS con una ruta incorrecta, o aplicar un estilo a una clase que no existe en el HTML.

# Taller: crea index.html y observa los cambios en directo

**Recordatorio teórico** Un archivo HTML es el documento base de cualquier página web. Su extensión es .html y su nombre más común —y con un significado especial para los servidores web— es index.html: cuando un servidor recibe una petición apuntando a una carpeta, busca automáticamente ese archivo como punto de entrada. Dentro del archivo, la estructura mínima válida incluye la declaración , las etiquetas , y . El contiene metadatos (título de la pestaña, codificación de caracteres con , etc.) que el navegador necesita pero no muestra directamente al usuario. El contiene el contenido visible. Un error frecuente en principiantes es confundir 'guardar el archivo' con 'actualizar el navegador': son dos acciones distintas. Si guardas en VS Code pero no recargas el navegador, verás la versión antigua. Otro error habitual es crear el archivo con extensión .txt en lugar de .html, lo que impide que el navegador lo interprete correctamente. Modelo mental clave: piensa en el flujo editor → guardar → navegador → actualizar como un ciclo de retroalimentación inmediata. Cada cambio que haces en el código debe seguir ese ciclo para verse reflejado.
